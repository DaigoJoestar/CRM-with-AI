1. Project Overview
Objective: Build a technical proof-of-concept for an "AI-First" CRM Log Interaction screen specifically for Healthcare Professionals (HCPs).

Core UX: A hybrid interface featuring a conversational chat on one side and a structured React form on the other, synchronized via Redux.

The "AI-First" Mandate: The user should be able to log a complex medical sales call entirely through natural language, with the AI handling extraction, validation, and database entry.

2. Technical Stack
Frontend: React (Vite), Redux Toolkit (State Management), Google Inter Font.

Backend: FastAPI (Python), LangGraph (Agent Orchestration).

AI Models (via Groq): * gemma2-9b-it: Primary model for entity extraction and tool calling.

llama-3.3-70b-versatile: Secondary model for complex medical context and summarization.

Database: PostgreSQL or MySQL.

3. Data Schema & State Management
The system must track the following "Interaction State":

hcp_id: UUID of the physician.

hcp_name: Full name and specialty.

interaction_date: ISO timestamp.

discussion_topics: List of medical products or clinical trial data discussed.

sentiment: Assessment of the HCP's reception (Positive/Neutral/Concerned).

compliance_check: Boolean flag indicating if the notes are free of off-label claims.

next_steps: Structured object for follow-up tasks.

4. LangGraph Agent Architecture
The agent is a stateful graph that processes user input (text or audio-transcription) and decides which tool to execute.

Required Tools (Python functions)
log_interaction:

Input: Unstructured text.

Process: Extracts entities via gemma2-9b-it, maps to schema, saves to DB.

edit_interaction:

Input: Interaction ID + Change Request.

Process: Updates specific fields in the DB while maintaining an audit trail.

compliance_validator:

Process: Scans notes for high-risk keywords (e.g., "off-label", "guaranteed results") before committing.

medical_context_provider:

Process: Fetches product info from a mock knowledge base to enrich interaction summaries.

nba_suggester:

Process: Uses llama-3.3-70b to suggest the "Next Best Action" for the sales rep based on HCP sentiment.

5. UI/UX Functional Requirements
Redux Sync: Any change made by the AI in the LangGraph state must trigger an immediate update in the Redux store to reflect in the React form fields.

Log Interaction Screen:

Left Panel: Real-time Chat (Groq-powered).

Right Panel: Structured Form (Google Inter typography).

Validation: Fields highlight in red if the AI has not yet extracted enough information.

6. Implementation Rules
Strict Constraint: Zero human-written code. All code must be generated by LLMs (Gemini/ChatGPT).

Compliance: Interaction summaries must be "Life Science Ready" (professional, clinical tone).

Error Handling: If gemma2-9b-it fails to extract a required field, the agent must ask a clarifying question rather than hallucinating data.
